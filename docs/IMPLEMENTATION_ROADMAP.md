## 三家公司的生產步驟預設配置

### DIN — 椰果加工
```
Step 1: 壓倍 (din_press)
  - 溫度 (°C)
  - 壓力 (bar)
  - 時間 (分鐘)

Step 2: 復水 (din_rehydrate)
  - 水溫 (°C)
  - 浸泡時間 (小時)
  - 水質

Step 3: 切割 (din_cut)
  - 切割規格 (8mm / 10mm / 5mm)
  - 切割速度
  - 刀具狀況
```

**生產表單欄位：**
- 溫度 (°C) | 壓力 (bar) | 時間 (分鐘)
- 復水時長 (小時) | 水溫 (°C)
- 切割規格 | 品質評級 (優/中/差)
- 備註

### SJA — 珍珠奶茶原料
```
Step 1: 混合 (sja_mix)
  - 配方版本
  - 混合溫度 (°C)
  - 混合時間 (分鐘)

Step 2: 包裝 (sja_package)
  - 包裝規格 (1kg / 5kg / 10kg)
  - 包數量

Step 3: 檢驗 (sja_qc)
  - 檢驗結果 (通過/不通過)
```

**生產表單欄位：**
- 配方版本 | 珍珠粉 (kg) | 塊根粉 (kg) | 調味料 (kg)
- 混合溫度 (°C) | 混合時間 (分鐘)
- 包裝規格 | 包數量 | 檢驗結果
- 備註

## 依賴關係

```
生產記錄 (production_records)
    ↓ 完成後自動
入倉 (item_batches) ← 庫存扣減
    ↓ 選擇批號
出貨單 (shipments) → 出貨明細 (shipment_items)
    ↓ 確認出貨
庫存減少 (item_variants) ← 扣減
    ↓
完整追蹤歷史記錄
```

## 風險&緩解

| 風險 | 影響 | 緩解 |
|------|------|------|
| 表單欄位過多導致維護困難 | 系統複雜度高 | 先用固定欄位，後支持動態 |
| 批號重複 | 數據混亂 | 用 UUID + 公司代碼確保唯一 |
| 庫存不同步 | 數據不一致 | 所有操作使用交易 (transaction) |
| 員工權限混亂 | 數據洩漏 | 基於公司和角色的行級安全 (RLS) |
